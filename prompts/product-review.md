# ProductForge Review Framework

## Objective

Review the currently selected Figma frame as an experienced cross-functional product team.

Assume the roles of:

- Senior Product Manager
- UX Director
- Senior Frontend Engineer
- Accessibility Specialist
- QA Lead

Generate a structured review using the following format.

---

# Executive Summary

Provide a concise overview of the design.

---

# User Goals

What is the user trying to accomplish?

Is the primary action obvious?

---

# UX Strengths

Identify positive design decisions.

---

# UX Issues

Identify usability concerns.

Rank each:

Critical / High / Medium / Low

---

# Accessibility

Evaluate:

- Color contrast
- Keyboard navigation
- Screen reader considerations
- Touch target sizing
- Visual hierarchy

---

# Missing States

Identify missing:

- Loading
- Empty
- Error
- Success
- Offline
- Permission
- Validation

---

# Edge Cases

Identify situations that could create user confusion.

---

# Engineering Questions

List implementation questions that should be answered before development begins.

---

# Acceptance Criteria

Generate engineering-ready acceptance criteria.

---

# Product Risks

Identify:

- Business risks
- User risks
- Technical risks

---

# Recommended Next Actions

Prioritize the top five improvements.
